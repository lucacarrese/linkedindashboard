# LinkedIn Post Frameworks

## Post Anatomy
Every post follows this structure:
1. **Hook** — Line 1. The only job is to stop the scroll.
2. **Setup** — 1-2 lines of context or contrast to pull the reader in
3. **Body** — The value. Use bullets, numbered steps, or short paragraphs.
4. **Landing line** — A punchy one-liner that lands the key insight
5. **CTA / Question** — Invite engagement. One sentence.

---

## Hook Types (use one per post)

### 1. Contrarian Statement
Challenge a common belief.
> "Most teams are using AI wrong."
> "Clay is not your outbound problem."
> "Claude Code is hyped for the right reasons."

### 2. Specific Result
Lead with a number or outcome.
> "153 contacts. Enriched emails. Live campaign. 20 minutes."
> "We replaced $300/month in tools with one Claude Code agent."

### 3. Bold Claim
State something strong that needs explanation.
> "Our outbound agents don't sleep."
> "We don't touch LinkedIn's campaign manager anymore."

### 4. Relatable Pain
Name a frustration your audience knows well.
> "Building LinkedIn ad campaigns used to take 30+ minutes of fighting the UI."
> "Your outbound stack has 6 tools. None of them talk to each other."

### 5. List Promise
Promise a specific, scannable list.
> "3 GTM workflows you can deploy with Claude Code today."
> "Here's the exact 5-step outbound flow we built inside Claude Code."

## Post Formats

### The How-To (Step-by-Step)
Best for: workflows, processes, tutorials
Structure: Hook → Problem → Numbered steps → Result → CTA

### The Use Case
Best for: specific tools, features, or integrations
Structure: Hook → What it is → How it works (bullets) → Key result → CTA

### The List Post
Best for: multiple examples, options, or frameworks
Structure: Hook → Intro line → 3-5 bulleted items with brief explanation → Landing line → CTA

### The Insight / POV
Best for: industry observations, mindset shifts, contrarian takes
Structure: Hook → Setup → Explanation with evidence → What it means for the reader → CTA

### The Story
Best for: personal experience, case studies, before/after
Structure: Hook → Context → What happened (chronologically) → Key insight → CTA

---

## Post Length Guidelines
- **Short** (under 150 words): POV posts, contrarian takes, single insights
- **Medium** (150-300 words): Use cases, how-tos with 3-5 steps
- **Long** (300-500 words): Multi-part lists, detailed workflows, case studies

Aim for medium by default. Long posts only when the detail adds real value.
