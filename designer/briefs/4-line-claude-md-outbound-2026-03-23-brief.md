# Design Brief — 4-line-claude-md-outbound-2026-03-23

## Post Content
```
Our entire outbound operation runs from a 4-line instruction file.

Here is what it says:

→ Score companies using this ICP criteria document
→ Find contacts via Apollo API
→ Write copy using our best-performing frameworks
→ Create and push campaigns to Instantly

That is it. Four lines. No code. No technical setup.

From here, Claude handles everything else:
→ Reads the API docs for each tool
→ Connects via API keys you provide
→ Decides what endpoint to use at each step
→ Batches work at 100-200 records to maintain quality
→ Saves all outputs so every campaign builds on the last

The file is called CLAUDE.md.
It is the brain behind the whole system.

Most people think automating outbound at this level requires an engineering team, expensive tooling, or months of setup.

It requires four lines of plain English and an afternoon.

What would your 4-line instruction file say? 👇
```

## Hook (first line)
> Our entire outbound operation runs from a 4-line instruction file.

## Tone
Bold/Authority

## Layout Type
Stat Card

## Key Visual Message
Simplicity as power — 4 lines of plain English running an entire outbound operation.

## Dominant Element
The 4 lines rendered large and clearly, like a code block or document card — minimal, stark, powerful

## Color Mood
dark

## Notes for Designer
Show the 4 lines as if they are in a real file/code block — monospace font, subtle border, clean. Make it feel like you are looking at the actual CLAUDE.md. The contrast is: this tiny file → entire outbound system. That contrast should land visually.
