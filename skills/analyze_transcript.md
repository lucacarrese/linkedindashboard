# Skill: Analyze Transcript → LinkedIn Posts

## When to Use
Use this skill when given a webinar, podcast, interview, or meeting transcript as source material.

## Step-by-Step Process

### Step 1 — Extract Raw Material
Read the full transcript and pull out:
- Main topics discussed (in order)
- Specific use cases, workflows, or systems mentioned
- Concrete results, numbers, or metrics
- Quotes or one-liners that stand out
- Tools and integrations mentioned

### Step 2 — Identify Post Ideas
For each use case or insight, ask:
- Is there a specific result or number to anchor this?
- Can this be turned into a step-by-step process?
- Is there a contrarian or surprising angle?
- Would a GTM professional find this immediately actionable?

Prioritize ideas that are **specific**, **results-driven**, and **actionable**.

### Step 3 — Match to Post Format
For each idea, select the best format from `/frameworks/post_structure.md`:
- Workflow with clear steps → How-To format
- Single tool or integration → Use Case format
- Multiple examples → List Post format
- Mindset shift or observation → Insight/POV format

### Step 4 — Draft Posts
Before writing, read `/examples/_patterns.md` for structure rules and pick 1-2 matching examples from `/examples/` to mirror the rhythm and voice.

Follow rules in `/context/audience_and_tone.md`:
- One idea per post
- Hook first, always
- Bullet points for lists of 3+
- End with a question

### Step 5 — Save Output
Save each post to `/posts/` with a filename format:
`[topic]-[format]-[date].md`
Example: `linkedin-ad-builder-usecase-2026-03-03.md`

---

## Quality Checklist
Before finalizing any post:
- [ ] Does the hook stop you from scrolling?
- [ ] Is there at least one specific number or result?
- [ ] Are all lists formatted as bullets?
- [ ] Is every paragraph 3 lines or fewer?
- [ ] Does it end with a question?
- [ ] Does it have one clear idea — not two or three?
