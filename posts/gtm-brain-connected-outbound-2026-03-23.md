# GTM Brain: Connected Outbound — 2026-03-23

Clay. Apollo. Instantly. 3 tools running your outbound.
None of them talking to each other.

Apollo has your contacts. Clay has your enrichment. Instantly has your sequences.
Same data. Siloed everywhere. Every campaign starts from scratch.

We fixed this with a single Claude Code setup.

Here is how it works:

1️⃣ **One folder. All context.**
Company lists, contact data, copy frameworks, campaign analytics — saved locally. Claude reads all of it before doing anything.

2️⃣ **One instruction file. All behavior.**
A CLAUDE.md file tells Claude: use this scoring criteria, find contacts via Apollo, write copy from these frameworks, push to Instantly. Four lines. Nothing more.

3️⃣ **Every campaign makes the next one better.**
Claude connects to your Instantly account, pulls best-performing campaigns by analytics, extracts the copy frameworks, and builds on them automatically. The system compounds.

What you end up with is a single brain that knows your ICP, your tools, your copy, and your results — all in one place.

Old way: siloed tools, manual handoffs, starting from scratch every campaign.
New way: one context layer, everything connected, outputs that improve over time.
