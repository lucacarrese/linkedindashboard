# Example 11 — Numbered Education List (10 Tips)
**Format:** Behavioral Shift Hook → 10 Numbered Points (Bold Title + 2-sentence explanation) → P.S. CTA
**Hook type:** Personal admission + promise ("I spend more time in X than Y")
**CTA type:** P.S. format (soft, feels like an afterthought)

---

I now spend more time in Claude Code than Clay.

Here are 10 things every GTM operator needs to know to get started:

1. You don't need to know how to code

VS Code + Claude Code extension. Install, sign in, start prompting. If you can use ChatGPT, you can use Claude Code.

2. Always create a CLAUDE.md file first

This is your system prompt for the project. Without it, Claude is generic. With it, it becomes an expert in your workflows, tools, and data.

3. Use the WAT Framework

Workflows (markdown instructions) → Agent (Claude Code) → Tools (scripts that execute actions). Three folders. Clean and scalable.

4. Plan Mode before you build anything

Don't just prompt and let it rip. Plan Mode makes Claude think harder, ask better questions, and catch gaps you missed.

5. Over-communicate your first prompt

Describe the goal AND the features. Claude will ask you questions you hadn't thought of - that's the whole point.

6. MCP Servers are your app store

Connect to Gmail, Sheets, Slack, your CRM - once. Claude handles all the endpoints and parameters automatically.

7. Skills = reusable knowledge packs

Install skill files for design, n8n, data enrichment - whatever you need. Claude loads only what's relevant per task.

8. Claude fixes its own mistakes

Hits an error → investigates → fixes the code → updates the workflow so it never breaks again. Self-healing.

9. Deploy to Modal for ~$0

Push your workflow to the cloud. Pay only when it runs. Schedule triggers or webhook triggers. Runs while you sleep.

10. Run a security review before deploying

One prompt: "Check for exposed API keys and vulnerabilities." 30 seconds. Prevents a nightmare.

You don't need to be a developer to build automations that used to require one.

Start with one workflow. Improve it. Deploy it. Then build the next one.
